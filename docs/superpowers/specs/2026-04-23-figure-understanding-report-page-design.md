# 图像理解层静态报告页设计

## 背景

当前图像理解层已经能把真实论文中的 figure 抽成结构化记录，但这些结果主要落在 `jsonl`、`csv`、`manifest` 和本地图片目录里。对非工程视角来说，这种输出不直观，尤其是 `figure_type`、`recaption`、`figure_summary` 和 `needs_manual_review` 需要对照图片一起看，才能判断识别是否合理。

## 目标

做一页本地静态 HTML 报告，让用户可以直接在浏览器里查看图像理解层的最终效果。

这页报告必须同时展示：

- 原始裁图
- 结构化理解结果
- 人工复核状态
- 图像与文字说明的对应关系

## 方案

采用最短路径：生成一个单页静态报告，不做搜索、不做筛选、不做多页面跳转。

报告由一个本地脚本生成，输入是已有的图像理解输出目录，输出是可直接打开的 `index.html`。页面只读，不依赖前端框架，也不需要后端服务。

### 文件边界

- 输入：
  - `figures_v1.jsonl`
  - `figures_review.csv`
  - `manifest.json`
  - 真实 figure 图片目录
- 生成器：
  - `scripts/render_figure_understanding_report.py`
- 输出：
  - `reports/figure-understanding/<paper_id>/index.html`

## 页面结构

### 1. 顶部总览

页面顶部显示：

- 论文 ID
- 总图数
- 自动通过数
- 人工复核数
- 页面生成时间

这一区域只用于快速判断这篇论文的图像理解是否整体正常。

### 2. 图像卡片区

每张图渲染成一张卡片，卡片内左右分栏：

- 左侧：裁图预览
- 右侧：结构化结果

右侧展示字段：

- `figure_id`
- `page_no`
- `figure_type`
- `confidence`
- `needs_manual_review`
- `caption_text`
- `recaption`
- `figure_summary`
- `panel_labels`
- `subfigure_map`

图片点击后打开原图大图，方便逐张检查。

### 3. 状态提示

卡片上用醒目标记区分：

- 自动通过
- 人工复核

人工复核项保持明显可见，不隐藏。

### 4. 边界说明

页面底部固定显示三类说明：

- 哪些图算成功
- 哪些图被保留人工复核
- 哪些是已过滤的 page 1 噪声图

## 视觉方向

页面采用“研究档案 / editorial dossier”风格，而不是通用仪表盘风格。

特征：

- 深色标题 + 浅色纸面背景
- 清晰的图卡边框
- 自动通过与人工复核使用不同的状态色
- 元信息采用更紧凑的标签样式
- 页面整体强调“看图”和“对照结论”，不是强调分析图表

## 数据规则

页面只读取已经生成的图像理解结果，不在浏览器端重新推理。

渲染时必须保证：

- 每条记录都能找到对应图片
- 图片路径和 JSONL 记录一一对应
- `needs_manual_review=true` 的记录仍然要显示出来
- 缺少 `caption_text` 的记录不应被自动隐藏

## 非目标

本期不做以下内容：

- 搜索
- 筛选
- 分页
- 多论文聚合
- 在线服务
- 重新识别图像

## 验收标准

这页报告满足以下条件才算完成：

- 能打开本地 HTML 页面
- 页面上能看到真实裁图
- 每张图能看到对应的 `figure_type`、`recaption`、`figure_summary`
- 能一眼分辨自动通过和人工复核
- 与 `_tmp_matdes_figures2` 里的真实输出一致
