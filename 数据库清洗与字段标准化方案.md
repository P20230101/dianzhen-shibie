# 数据库清洗与字段标准化方案

## 1. 文档目的

本文档用于把你当前的 `Lattice crashworthiness data.xlsx` 从“可参考的人工汇总表”升级为“可持续扩展、可自动抽取、可用于检索与建模”的标准化数据库。

目标不是简单整理 Excel，而是建立一套后续可以支撑：
- 文献自动入库
- 多模态信息抽取
- 结构—性能检索
- 机器学习建模
- 逆向设计候选筛选

的统一字段体系与清洗规则。

---

## 2. 当前 Excel 的参考价值与局限

### 2.1 参考价值

你当前 Excel 具有很强的参考价值，因为它已经隐含了一个“点阵吸能样本级数据库原型”。

基于当前表格可观察到：
- 工作簿包含 2 个 sheet：`Equations`、`Database`
- `Database` 是核心数据表
- 有效数据约 **619 条样本记录**（除去表头）
- 当前主表约有 **27 个有效字段列**
- 字段已经覆盖了 6 类核心信息：
  - 文献信息
  - 结构设计
  - 材料属性
  - 制造工艺
  - 试验/分析条件
  - 力学/吸能性能

这说明你已经不是从零开始，而是已经拥有：
1. 初始 schema 雏形
2. 已整理的人工先验
3. 可反向指导自动抽取的 gold seed
4. 可用于后续评估抽取效果的对照基准

### 2.2 当前 Excel 的主要局限

当前表格仍然更接近“综述式整理表”，而不是“工程化样本数据库”。主要问题包括：

1. **字段语义混合**
   - 例如 `Design`、`Structure`、`Type` 三列都在描述结构，但粒度不一致。
   - `Material` 和 `Material.G` 也存在“具体材料名 / 材料大类”混用。

2. **缺失值表示不统一**
   - 当前同时存在 `-`、空白、` `、`None` 风格的缺失表达。

3. **值域不规范**
   - 例如：
     - `Truss` / `TRUSS`
     - `Bioinspired` / `Bioisnpired`
     - `General 2D` / `2D General`
     - `Aluminium` / `Aluminum`
     - `Ti-6Al-4V` / `Ti–6Al–4V`
     - `Nylon 12` / `Nylon-12` / `PA12` / `PA-12` / `Polyamide 12`

4. **单位与字段含义未完全分离**
   - 例如：`Density (g/cm³)` 中出现了明显与常识不一致的值风格，需要核验是否存在单位换算或原始录入问题。
   - `EA (J)` 与 `EA (J/m³)`、`SEA (J/g)`、`SEA (J/CM³)` 之间也需做统一与派生逻辑设计。

5. **证据链缺失**
   - 当前表中只有结果值，没有保存该值来自正文、表格、图还是推导。
   - 后续自动抽取和人工复核会受限。

6. **不适合直接做模型输入**
   - 因为结构名称尚未层次化，材料与工艺未标准化，性能字段的缺失模式也较重。

## 4. 清洗总原则

### 4.1 不丢原始信息
所有标准化都必须保留原始字段：
- `raw_design`
- `raw_structure`
- `raw_type`
- `raw_material`
- `raw_process`

标准化字段与原始字段并存，而不是覆盖。

### 4.2 先规范，再推断
- 先做字符串统一、单位统一、缺失值统一
- 再做结构主类识别、材料大类映射、性能派生
- 最后再做样本级推断或功能检索

### 4.3 优先保持“可追溯”
每个自动修正都应可回溯：
- 原值
- 规则
- 修正值
- 是否人工确认

### 4.4 缺失值不要伪填充
对确实没有报道的数据，统一标为 `null`，不使用 `-`、`N/A`、空格混杂表达。

### 4.5 抽取优先级要前置到 schema 中
后续数据库应支持以下来源优先级：
- `T1_text_table`：正文/表格直接抽取
- `T2_derived`：由已知量推导
- `T3_figure_reading`：图中读数
- `manual_review`：人工补录

## 6.3 缺失值统一规则

### 所有以下值统一转为 `null`
- `-`
- 空白单元格
- 仅空格 ` `
- `None`
- `N/A`
- `not reported`
- `not mentioned`

### 注意
不能把近似值误清洗掉，例如：
- `≈ 58`
- `~58`
- `58 ± 3`

这些应进入“数值解析层”，不能直接当缺失。

- `voronoi`

### 第二层：`structure_subtype_list`
允许数组，不强制单值。因为很多样本同时带有：

### 第三层：保留原始值

## 6.9 数值字段清洗规则

### 通用规则
所有数值字段增加两列：
- `*_raw`
- `*_value`

例如：
- `plateau_stress_raw`
- `plateau_stress_mpa`

### 数值解析规则
- `58` -> 58.0
- `≈ 58` -> 58.0，另记 `value_approx=true`
- `~58` -> 58.0，另记 `value_approx=true`
- `58 ± 3` -> 主值 58.0，另存误差 3.0
- `58-62` -> 可拆 `value_min=58` `value_max=62`

### 当前优先处理字段
- relative_density_raw
- youngs_modulus_gpa
- density_g_cm3

## 7. 哪些列适合删除，哪些列不建议删除

## 7.1 当前阶段不建议删除的列
即便重复，也不建议现在删除：
- Design
- Structure
- Type
- Material
- Material.G

原因：
这些列虽然混乱，但正是建立标准化映射表的关键来源。

## 7.2 当前阶段可不作为主分析字段的列
以下字段暂不建议作为机器学习主输入，但应保留：
- Authors
- Title
- DOI

原因：
它们更多用于追溯、查重与证据链接，不是结构—性能建模主特征。

## 7.3 后续可能降级为派生列的字段
- Material.G
- Efficiency EA (%)
- SEA (J/CM³)

这些字段后续可保留，但不一定作为核心字段。

## 8. 推荐的清洗执行顺序

### 第 1 步：建立 raw 备份表
导出一份 `database_raw_backup.csv`，不做任何修改。

### 第 2 步：统一缺失值
把 `-`、空白、空格统一成 `null`。

### 第 3 步：修复明显拼写和格式问题
例如：
- TRUSS -> Truss
- Bioisnpired -> Bioinspired
- Concentraded Load -> Concentrated Load
- Ti–6Al–4V -> Ti-6Al-4V

### 第 4 步：建立结构映射表
单独维护：
- `structure_mapping.csv`
  - raw_design
  - raw_structure
  - raw_type
  - structure_main_class
  - structure_subtype_list
  - notes

### 第 5 步：建立材料映射表
单独维护：
- `material_mapping.csv`
  - raw_material
  - material_canonical
  - material_family

### 第 6 步：建立工艺映射表
单独维护：
- `process_mapping.csv`
  - raw_process
  - process_canonical
  - process_family

### 第 7 步：解析数值字段
将文本型数值转成标准数值列，同时保留 raw。

### 第 8 步：生成 schema v1 主表
输出 `samples_v1.csv` 或 `samples_v1.parquet`。

### 第 9 步：加入 evidence 字段接口
即便当前 Excel 没有证据列，也预留：
- source_type
- source_page
- figure_id
- table_id
- extractor
- confidence

## 10. 与后续自动抽取系统的接口设计

为了让后续 PDF 自动抽取系统可直接入库，建议样本主表至少兼容：

- 论文级主键：`paper_id / doi`
- 样本级主键：`sample_id`
- 字段级来源：`source_type`
- 置信度：`confidence`
- 原始字符串：`*_raw`
- 标准值：`*_canonical` 或 `*_value`

这样后续自动系统抽出的字段可以直接 append 进数据库，而不是每次手动改 Excel。

## 11. 本阶段最重要的结论

当前 Excel 不是“最终数据库”，但它已经是非常有价值的：

1. **它是你的 schema 原型**
2. **它是自动抽取系统的先验模板**
3. **它是后续评估集和人工复核入口**
4. **它决定了后续系统应该抽哪些字段**

因此，正确做法不是推翻它，而是：

**保留原始字段 -> 建立标准字段 -> 增加证据字段 -> 输出样本级标准表。**
